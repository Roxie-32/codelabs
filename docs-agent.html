Introduction

As developers, we all know the frustration of sifting through endless pages of documentation, trying to find a single snippet of code or a specific endpoint detail. Manually scrolling, searching, and piecing together information is time-consuming and inefficient.

This is where AI agents become incredibly valuable. In this workshop, you will learn to build an interactive documentation agent that users can query directly to get more information about your API. Under the hood it uses Retrieval-Augmented Generation (RAG), which means it searches your own docs first and only then composes an answer based on what it found.

Instead of a tedious search, you can simply ask, "How do I authenticate X feature?" and the agent will instantly provide a comprehensive, well-formatted answer, saving you countless hours and streamlining your workflow. It also shows where the answer came from so you can trust it or jump to the full page. Think of it as a fast front door to the documentation you already have—not a replacement.

What you'll learn

  • Understand the core concepts of RAG and how it enhances the reliability and accuracy of large language models (LLMs)
  • Learn how to craft sophisticated prompts that guide an LLM to generate highly structured, detailed, and professional-looking content
  • Discover how to use Vertex AI Search as a powerful retrieval tool to find and extract relevant information from unstructured data, such as API documentation
  • Learn to integrate one of the world's most capable and multimodal models, Gemini, into your application pipeline

What you'll need

  • A Google account
  • A Google Cloud project with billing enabled
  • A web browser

What you'll build

You will build an AI-powered documentation agent that can easily answer any questions about your API. For the workshop, we'll run it as a shareable Gradio web app in Google Colab for easy access. Afterward, you can point it at your own docs and embed it in your documentation site—as a standalone page, an in-page widget, or even wired into your existing search bar.

Behind the scenes: Your agent is a sophisticated RAG pipeline that orchestrates intelligent retrieval using Vertex AI Search, augments prompts with retrieved information, and generates high-quality responses using Gemini on Vertex AI.

Set up your Google Cloud environment

Before writing any code, you need to set up your Google Cloud environment. This ensures your project has the correct permissions and APIs enabled to access the necessary AI services.

1. Select a Google Cloud Project

In the Google Cloud Console, use the project selector to either choose an existing project or create a new one. Remember, your project is the central hub for all the resources you'll use in this workshop.

2. Enable Billing

Ensure that billing is enabled for your project. This is a non-negotiable step, as most of the services we'll use (Vertex AI Search and Gemini) are not part of the free tier. Visit the official guide to learn how to check if billing is enabled on a project.

Note: Skip this step if you're using cloud credits.

3. Enable Required APIs

Navigate to APIs & Services. Search for and enable the following APIs:

  • Vertex AI API: This allows you to use the Gemini model
  • Discovery Engine API: This is the engine that will power your document search
Tip: We've listed all the necessary APIs here to prevent issues later on. Make sure both are enabled before proceeding.

Upload your API docs to Cloud Storage

First, we need to gather all the documents you want your agent to know about. For this workshop, we'll be using the popular Petstore API as our example.

Download the workshop files

  1. Start by downloading the documentation directory for this fictional API. The docs directory contains an openapi.json file - A JSON file containing a list of the API's endpoints.
  2. Unzip the folder you downloaded so you can have access to the openapi.json file

Create a Cloud Storage (GCS) Bucket

To create a Cloud Storage bucket:

  • In the Google Cloud Console, search for Cloud Storage and click Create bucket
  • Give your bucket a globally unique name (e.g., my-api-docs-workshop-2025). Click on Continue
  • In the Location Type, select Multi-Region. In the drop down, make sure to select the option us (multiple regions in United States)
  • Leave the rest of the options at their default settings and click Create bucket
  • If you're prompted to Prevent public access, click Confirm

Upload your documentation

Now, upload the files you just downloaded into your new bucket.

  • Once your bucket is created, click on its name to navigate inside
  • Click the Upload folder button and select the openapi.json file you downloaded
Success! Your API documentation is now securely stored and ready to be processed into a searchable knowledge base.

Create a Vertex AI data store

To create a searchable knowledge base, we need to ingest the documents from your Cloud Storage bucket into a Vertex AI Data Store. This is the core of our RAG pipeline's "retrieval" step.

Why Vertex AI Search? By creating a Data Store within Vertex AI, your documents are automatically processed, indexed, and converted into a format that allows for semantic, natural-language search. This means your agent can find relevant information based on meaning, not just keywords, which is much more effective.

Enable AI Applications

  1. In the Google Cloud Console, search for AI Applications in the search bar and navigate to the page
  2. You'll be prompted to activate the API. Click Continue to proceed

Configure your data store

  1. Once on the AI Applications page, click on Data Stores in the left-hand navigation menu
  2. Click Create data store
  3. For the Source page, select Cloud Storage as your data source
  4. Click Browse and navigate to your docs folder inside the GCS bucket you created in the previous step, then Select the openapi.json file. Alternatively, you can enter the full path directly in the gs:// field
  5. Select Unstructured documents (PDF, HTML, TXT and more) as the kind of data you are importing. This tells Vertex AI to intelligently parse and extract text from your files, even from complex formats like JSON
  6. For Synchronization frequency, select One-time. This is sufficient for the workshop since we don't expect the documentation to change
  7. Click Continue. Leave the Location as global for this workshop to ensure maximum availability of all search and answering services
  8. Enter a name for your data store (e.g., petstore-docs-agent)
  9. Click Create
Note: Vertex AI Search will then start processing your files. This might take a little while depending on how many you have. To check the ingestion status, go back to the Data Stores page and click your data store's name. You can monitor the progress on the Activity tab. The process is complete when the status changes from "In progress" to "Import completed".

Build the Agent

Now that you've set up your environment and ingested your data, you're ready to build the full RAG agent. This final step is where all the pieces come together to provide a seamless, interactive experience.

You'll be building the agent using Python and Gradio. You'll perform all the steps within a single Google Colab notebook. If you've never used Google Colab, don't worry—it's incredibly simple. Think of it as a shared notebook that runs entirely in your browser. All the code and commands have been prefilled, so all you have to do is run the code cells by clicking the play icon next to them.

Before you proceed, have the following credentials handy as you'll need them for the notebook's configuration:
  • PROJECT_ID - The ID of your Google Cloud project
  • LOCATION - The location of your data store bucket
  • DATA_STORE_ID - Your data store ID

Access the Colab Notebook

  1. Click the link below:
    Lagos Cloud Roadshow Colab for Docs Agent
  2. Once the notebook opens, navigate to File and click Save a copy in Drive. You'll be using your copy of the notebook to build the agent

What to expect in the Colab notebook

The notebook guides you through three simple steps to launch your agent. Just click the play button for each code cell.

1. Authentication

The first cell authenticates your Colab environment with your Google Cloud account. A new window will pop up asking you to log in and grant the necessary permissions. This step ensures the notebook can access your project's services.

2. Dependencies

The second cell installs all the required Python libraries for the project. This includes google-cloud-discoveryengine and google-cloud-aiplatform to interact with Google Cloud's AI services, as well as gradio to build the user interface.

3. Run the Agent

The next cells contain all the core logic:

  • Configure your credentials: You will update the PROJECT_ID, LOCATION, and DATA_STORE_ID variables
  • Execute the RAG pipeline: The bulk of the code contains the core logic that orchestrates the entire RAG process. It's a series of functions that seamlessly work together: they take a user's question, retrieve relevant information from your data store, augment a prompt with that information, and then send it to the Gemini model to generate the final, comprehensive answer
  • Launch the UI: A live chat interface will appear directly within the notebook, and you can begin asking questions about your API documentation

How it will work

Once you have successfully run the final code cell in the Colab notebook, a web-based chat interface will appear directly below it. This is your fully functional documentation agent. Here is what you can expect when you interact with it:

  1. Ask a Question: Type a natural-language question into the text box, just as you would in a regular chat application. You can use the provided examples to get started
  2. Wait for the Agent: When you hit enter or click the "Ask" button, the button will change to a "Loading..." state. This indicates that your request has been sent to the backend and the RAG pipeline is actively retrieving information and generating a response
  3. Receive the Answer: After a few seconds, a detailed answer will appear in the main chat window. The response will be professional and well-structured, following the specific format we engineered in the prompt
